# Project Workflow

## General Workflow

Before coding:
1. Understand the feature request
2. Explain the implementation plan
3. Suggest folder structure if needed
4. Identify reusable components
5. Consider responsiveness
6. Consider accessibility
7. Test locally
8. Deploy

## Development Rules

- Build features step-by-step
- Keep business logic separated
- Refactor repetitive code

## UI Workflow

When building UI:
- Prioritize layout structure first
- Then responsiveness
- Then styling polish
- Then animations

## State/Data Workflow

- Use TanStack Query for API/server data
- Handle loading states
- Handle empty states
- Handle error states
- Use query keys properly

## Debugging Workflow

When debugging:
- Explain the root cause
- Explain why the issue happened
- Suggest best-practice fixes
- Avoid rewriting unrelated code

## Learning Workflow

When introducing advanced concepts:
- Explain simply
- Explain why it is used
- Compare alternative approaches
- Keep explanations beginner-friendly where possible
